Is programming easy

Is programming easy?

Yup. If you use frameworks, I think it is easy to come up with an API and then build the frontend using your favorite frontend javascript framework.

For me, the most easy-to-understand framework for frontend is Angular.

You need a framework

Yes. My explanation is simple: Frameworks make projects more easier to build with in the immediate time-frame. As the project size goes larger, like you become a unicorn startup like what silicon valley raves about, my thinking goes you should have enough profits coming in that it’s worth it to invest in performance improvements.

Also, from a developer perspective, premature optimization is the root of all unmaintainability. For a genius programmer, probably it’s not a problem, but for the more non-brilliant programmers - it’s an issue as you frequently need to share how that part of the code works.

Programmers like to be fast and furious

The tool, Angular-CLI , is pretty fantastic starter kit for building Angular apps. While it takes a while for npm download the dependencies, it has a built-in component that you can just edit and it just works.

React also has a create-react-app cli tool to quickly get started on React. Same with vue, there’s the vue-cli and ships with some nifty Vue CLI plugins which help setup the build configuration to your liking.

With the advent of stackoverflow, google, and awesome docs, It’s so easy for one to just peruse how things work. If you still don’t understand the concept, well one can try youtube.

For me, it’s all about excellent tooling and documentation. It’s much easier today to start a career in web development than it was 10 years ago.
